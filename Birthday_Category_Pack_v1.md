# Birthday — Category Pack v1
> **Incremental spec on top of Template_Creation_Framework_v20.md**
> This document only covers what CHANGES or is NEW for the Birthday category.
> Everything not mentioned here (archetypes, layouts, tracks, colour palettes, font pairings, background finishes, render styles, prompt builder architecture) is inherited unchanged from the framework.

---

## 1. What This Category Is

Birthday is a celebration-first greeting category. Visual energy (balloons, cake, confetti, sparkle) shares the canvas with the greeting text — "Happy Birthday" replaces "Good Morning" as the hero text.

**Only two things change from Good Morning:**

1. **Border motifs / A2 accent motifs** — replaced with Birthday-specific motif sets per track
2. **Illustration catalogue** — replaced with Birthday-specific items (30 new + 4 carry-overs)

Everything else — wizard flow, archetypes, tracks, layouts, background finishes, render styles, prompt builder architecture — carries over unchanged.

---

## 2. Wizard Flow — No Change

**Birthday flow is identical to Good Morning:**
`Archetype → Track → Layout → Colour → Font → Illustration → Intensity`

No Feeling step. Birthday has one emotional lane: celebration. The track system (A–E) provides all the tonal variation needed — from maximal party (Track A) to soft warmth (Track E).

All 5 tracks remain available for every Birthday template.

## 5. Layout Bias (Guidance, Not Hard Filter)

| Layout | Birthday Suitability | Reason |
|--------|:-------------------:|--------|
| L5 — Text Bottom | ★ Best | Illustration/cake fills top, "Happy Birthday" anchors at base — classic card |
| L2 — Overlay / Text Centre | ★ Best | Atmospheric party background with centred greeting overlay |
| L1 — Text Top | ✓ Works | "Happy Birthday" at top, illustration below — clean |
| L6 — Text Centre (frame) | ✓ Works | Strong for A1 — frame with centred greeting, corner illustrations |
| L3 / L4 — Text Left / Right | ✓ Works | Birthday illustrations are compact enough for side placement at any size |

**Note:** Unlike Suvichar (which cautioned L3/L4), Birthday works well with side layouts because greeting text is short and illustration subjects (cake, balloons) have clear focal points.

## 6. Prompt Framework — Birthday Context

The prompt builder architecture (functions, assembly order, platform adapters) is **inherited unchanged** from the framework. What changes are the **context variables** that feed into each builder function. This section defines every variable swap.

### 6.1 Text Content

The greeting text changes from "Good Morning" to "Happy Birthday":

| Variable | Good Morning Value | Birthday Value |
|----------|-------------------|----------------|
| Default hero text | "Good Morning" / "शुभ प्रभात" | "Happy Birthday" / "जन्मदिन मुबारक" |
| Text character range | Short greeting (1–2 lines) | Short greeting (1–2 lines) — same range |
| Optional sub-text | Blessing / quote line | Birthday wish / blessing line |
| Text language options | Hindi, English, Hinglish | Hindi, English, Hinglish — no change |

**User input field:** The text input step shows "Happy Birthday" as default placeholder. User can customise with name: "Happy Birthday Priya!", "जन्मदिन की हार्दिक शुभकामनाएं", etc.

### 6.2 Scene Atmosphere & Mood

These feed into `buildSubjectLine()` and `buildStyleLine()`:

| Prompt Element | Good Morning | Birthday |
|----------------|-------------|----------|
| Scene atmosphere | `warm golden morning light, India morning atmosphere` | `festive celebration atmosphere, warm joyful lighting, birthday party ambience` |
| Mood keywords | `peaceful, serene, spiritual, new day` | `joyful, celebratory, festive, warm, happy, cheerful` |
| Time of day | `morning` (locked) | *(none — birthday is time-agnostic)* |
| Cultural context | `Indian morning ritual` | `Indian birthday celebration` |

### 6.3 Per-Archetype Subject Line Construction

The subject line is the core of the prompt. Here's how it assembles for each archetype in Birthday context:

**A1 (Text + Frame):**
```
[frame type] frame with [frame decoration level] [birthday motif set from track],
[illustration item] at [illustration size],
[palette colour description],
festive celebration atmosphere, warm joyful lighting,
[render style from track]
```
Example: `Full perimeter frame with detailed gold foil balloons and confetti burst motifs, birthday cake at medium size, warm gold and cream palette, festive celebration atmosphere, warm joyful lighting, 3D glossy render`

**A2 (Text + Illustrations):**
```
[illustration item] as [illustration size] focal element,
[density-calculated accent items from birthday motif set],
[palette colour description],
festive celebration atmosphere, warm joyful lighting,
[render style from track]
```
Example: `Balloon bunch as medium focal element, scattered confetti burst accents and sparkle cluster fills, pink and gold palette, festive celebration atmosphere, warm joyful lighting, watercolour botanical style`

**B1 (Full Photo Background):**
```
[illustration item / scene subject] [subject focus position],
festive celebration atmosphere, birthday party ambience,
single continuous full-bleed photograph, one unified scene filling the entire 9:16 canvas,
photorealistic photography, DSLR quality, natural lighting
```
Example: `Birthday cake with lit candles on a decorated table, upper-centre of the frame, festive celebration atmosphere, birthday party ambience, single continuous full-bleed photograph, one unified scene filling the entire 9:16 canvas, photorealistic photography, DSLR quality, natural lighting`

**B1 rules (inherited, restated for clarity):**
- No palette colour in prompt (causes decorative hallucination)
- No text space / zone instructions (`getTextSpaceInstruction` returns `''`)
- No overlay description in prompt (overlay = post-production design layer)
- Subject focus describes position only, no "leave space" language

### 6.4 Style Line

`buildStyleLine()` output per archetype:

| Archetype | Good Morning | Birthday |
|-----------|-------------|----------|
| A1 | `[render style], warm morning tones, [frame decoration level] ornamentation` | `[render style], festive celebration tones, [frame decoration level] ornamentation` |
| A2 | `[render style], warm morning tones, [density] accent arrangement` | `[render style], festive celebration tones, [density] accent arrangement` |
| B1 | `Photorealistic photography, DSLR quality, no CGI, no illustration, morning scene in India` | `Photorealistic photography, DSLR quality, no CGI, no illustration, birthday celebration scene, Indian home or party setting` |

### 6.5 Negative Terms

`buildNegativeTerms()` — Birthday-specific additions on top of existing per-archetype negatives:

| Archetype | Inherited Negatives (no change) | Birthday Additions |
|-----------|-------------------------------|-------------------|
| A1 | text, words, letters, watermark, blurry, photograph... | `funeral, mourning, sad, wilted, dark mood, horror` |
| A2 | text, words, letters, watermark, blurry, photograph... | `funeral, mourning, sad, wilted, dark mood, horror` |
| B1 | collage, composite, split panel, two photos, stacked, dual image... | `funeral, mourning, sad, wilted, dark mood, horror, illustrated elements, graphic design, decorative overlay` |

### 6.6 Text Space Instruction

`getTextSpaceInstruction()` — behaviour per archetype:

| Archetype | Behaviour | Change from Good Morning? |
|-----------|-----------|:---:|
| A1 | Returns layout-based text zone instruction (frame provides structure) | No change |
| A2 | Returns layout-based text zone instruction | No change |
| B1 | Returns `''` (empty — no text zone in photo prompt) | No change |

### 6.7 Platform-Specific Notes

All platform adapters (Midjourney, Firefly, Leonardo, Gemini) inherit unchanged. The only platform-facing changes are the variable values above flowing through the same assembly pipeline.

| Firefly | Same structure. "Morning" references replaced with "celebration" in scene description field. |
